Git Rebase Interactive Simulator

Rebase Actions Guide

  • Pick: Mantener el commit tal como está.
  • Reword: Mantener el contenido del commit pero editar el mensaje del commit.
  • Edit: Detener el rebase en ese commit para que puedas modificar los archivos o agregar más cambios y hacer un nuevo commit.
  • Squash: Combinar este commit con el anterior, integrando ambos cambios pero te permitirá editar el mensaje del commit.
  • Fixup: Similar a squash, pero descarta el mensaje de este commit y solo mantiene el mensaje del commit anterior.
  • Drop: Eliminar completamente el commit de la historia.

NOTA: Si clickas sobre un commit lo selecciona y si clickas sobre otro intercambia las posiciones
